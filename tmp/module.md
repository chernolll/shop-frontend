## 达人模块

### 4.1 达人筹备表（BD核心资源 · 重新设计）

筹备表与任务解绑，作为BD自主管理达人资源的私有空间。

#### 4.1.1 字段定义

| 字段          | 说明               | 备注                      |
| ------------- | ------------------ | ------------------------- |
| kol_id        | TikTok主页唯一标识 |                           |
| kol_url       | TikTok主页链接     |                           |
| has_budget    | 有无预算           | Boolean                   |
| budget_amount | 预算金额           | 仅 has_budget=true 时需填 |
| created_at    | 录入时间           | 系统自动记录              |
| remark        | 备注               | 可选                      |

#### 4.1.2 业务规则

1. 每个BD只能查看自己的筹备数据；
2. kol_id 可被多个BD录入，但仅第一条（最早时间）生效，其余标记为"重复"；若已被其他BD抢先录入，显示归属BD信息，供线下协商；
3. BD可移除自己筹备表中的达人；移除后，该kol_id在其他BD筹备记录中，按录入时间最早的自动生效；
4. BD可更改：有无预算、预算金额、备注；
5. 录入时联查 kol 表：若该达人存在且非正常状态，该筹备记录不生效。

#### 4.1.3 录入方式

- 支持下载模板 → 填写 → 批量上传解析入库
- 支持表单单个录入

### 4.2 达人池（新增）

达人池是无主BD、可被BD筹备的公共资源池。

#### 4.2.1 来源

- 已离职BD：Admin释放后，该BD筹备表的达人自动转入达人池
- Admin主动上传：字段与筹备表一致

#### 4.2.2 规则

- BD可从达人池抢占达人到自己的筹备表
- 注意数据库原子性操作（防止并发重复抢占）

#### 4.2.3 录入方式（Admin）

- 支持下载模板 → 填写 → 批量上传解析入库
- 支持表单单个录入

### 4.3 KOL 达人库

记录与公司完整走过SOP流程的达人（合作过的达人），与筹备表关联性弱。

- 不需要筹备BD字段
- 只记录当前所属BD
- SOP结束时自动写入/更新：若kol不在表内，自动插入；若已存在，更新所属BD为该SOP的BD
