# 角色与权限

## 角色定义

| 角色    | 枚举值  | 说明                             |
| ------- | ------- | -------------------------------- |
| 管理员  | `admin` | 系统管理、全量数据查看、审核审批 |
| BD 运营 | `bd`    | 达人运营、任务执行、数据看板     |

定义位置: `playground/src/consts/role.ts`

## 权限控制机制

### 路由层

- 每个路由 `meta.authority` 声明允许的角色
- 路由守卫 (`guard.ts`) 在登录后根据用户角色生成动态路由
- 未授权的路由页面强制跳转 403

### 菜单层

- 动态菜单由 `generateAccess()` 根据角色过滤生成
- 不可访问的路由不会出现在菜单中

### 指令层

- `@vben/access` 提供 `v-access` 指令用于组件级权限控制
- 启动时通过 `registerAccessDirective(app)` 注册

## 核心约束

- BD 只能查看和操作自己的达人库和任务
- Admin 可以查看全量数据但不能执行 BD 的 SOP 操作
- 角色由后端返回（`UserInfo.roles`），前端不做角色判断
- 权限码（`accessCodes`）可做更细粒度控制
